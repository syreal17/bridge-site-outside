## Rule #1 — The game should always be playable.

At every stage:
- You can walk
- The Dryad exists
- You can make camp
- Something small works better than before

Never spend months building systems without a playable loop.

## Rule #2 — Don't rush.

This is for fun, there's no such thing as a deliverable for this project.
Creating is fun, savor it. Take joy in bringing something new into the world.

## Rule #3 — Maintain the dev log.

Looking back at all that has been accomplished is only possible if there is a
log. Maintaining a dev log will help me not rush, as well.

## Rule #4 — Create the game for me, the first player.

Since AI is going to be a major part of the game, playtesting will likely be as
unpredictable as actually playing the game for the first time.

## Rule #5 — Give credit where credit is due.

Remember where I get resources, so credit can be given.
